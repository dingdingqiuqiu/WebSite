## 1. 理论基础

数组是存放在连续内存空间上的相同类型数据的集合。

需要两点注意的是

- **数组下标都是从0开始的。**
- **数组内存空间的地址是连续的**
- **数组的元素是不能删的，只能覆盖。**

正是**因为数组在内存空间的地址是连续的，所以我们在删除或者增添元素的时候，就难免要移动其他元素的地址。**

使用C++的话，要注意vector 和 array的区别，vector的底层实现是array，严格来讲vector是容器，不是数组。

在C++中，二维数组存储地址连续，但在java中不是 `3*4` 的连续地址空间，而是四条连续的地址空间组成！如下图所示：

## 2. 二分法

利用数组排列的有序性降低复杂度到O(logn)

其核心理念在于找寻最小可信目标区间`[left,right]`每次排除一半区间，快速收敛，直到

1. `nums[mid] == target`(找到) 
2. `left>right`（没找到）

```java
class Solution {
    public int search(int[] nums, int target) {
                int res = -1;
        int left = 0, right = nums.length - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (nums[mid] == target) {
                res = mid;
                break;
            } else if (nums[mid] < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        return res;
    }
}
```

## 3. 双指针

关键是明确好两个指针的作用

1. 一个指针做事，一个指针探路
2. 两个指针一起做事直到满足退出条件

**通过一个快指针和慢指针在一个for循环下完成两个for循环的工作。**

分为**同向双指针**和**相向双指针**

## 4. 滑动窗口

特殊情况下的双指针 

例如**lc209.长度最小的子数组**就是利用上一轮的结果快速得出以`right`为结尾的最小子数组

滑动窗口特点：**越长越合法**

因此使用`l`指针收缩边界，`r`指针“扩充”边界
